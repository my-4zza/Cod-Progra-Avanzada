## Guía de Versionado en este Repo
1. **NO se crean archivos nuevos:** Las mejoras se sobreescriben en los archivos originales.
2. **Cabeceras:** Cada archivo fuente debe actualizar su número de versión internamente.
3. **Commits:** Cada carga a GitHub debe seguir el formato: `opt: archivo v1.X.X -> v1.Y.Y`.
